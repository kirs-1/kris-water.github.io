### 为什么需要泛型?
> [!info]
> - 如果一段代码(接口,类,方法等),但是结构是相同的,,使用泛型可以解决代码冗余的问题。

###### 不使用泛型,下面的这个类就要定义多个,非常的冗余
```java
public class MyArrayList {  
    private int[] Array;  
    public void add (Object  element) {}  
    public Object get(int  index) {  
        return Array[index];  
    }  
    public void remove (int index) {}  
  
    public void main(String[] args) {  
        MyArrayList list = new MyArrayList();  
        // 这里面是可以随意插入类型的,因为Array是Object类型  
        list.add(1);  
        list.add("666");  
        // 因为存入的值是Object类型,取的时候只能取出Object类型,然后强制转型  
        int a = (int) list.get(1);  
        // 如果将上面的类型指定为int,那么就不需要强制转型了,但是如果有多个ArrayList要装多种类型的数据,  
        // 就要写多个MyArrayList类,这样会非常冗余  
    }  
}
```

##### 以下是使用泛型后的版本
```java
// NewMyArrayList<T>这里的<T>是代表这个类有一个泛型类型,<K,V>也可以写成多个,  
// 就是代表着一个类有多个泛型类型  
public class NewMyArrayList<T> {  
//    java的写法在标明数组类型的时候,会有以下的写法  
    private T[] Array;  
//    而不是这样的  
//    private int Array2[];  
//    java的写法可以避免一种问题就是,下面这种写法会容易引起歧义,容易解读成b也是一个int数组  
//    private int a[], b;  
//    所以java的原则就是类型在前  
//    private int[] a,b;  
    public void add(T element) {}  
    public T get(int index) {  
        return Array[index];  
    }  
  
    public static void main(String[] args) {  
//      使用泛型之后,我在实例化这个数组的时候,可以声明这个数组的类型,有<>(菱形操作符)的写法,  
//      等式右侧就可以根据左侧的类型做类型推断,就不需要再写一次类型了  
        NewMyArrayList<Integer> list = new NewMyArrayList<>();  
        list.add(1);  
        int a = list.get(1);  
  
        NewMyArrayList<String> list2 = new NewMyArrayList<>();  
        list2.add("a");  
        list2.get(1);  
    }  
}
```
